1. Crear por cada tabla de la base de datos, una carpeta con el nombre de la tabla y su archivo index.html
2. Crear dentro de la carpeta de php un archivo .php por cada tabla de la base de datos con el nombre de la tabla,
ejemplo: users.php
3. Se deberán de reutilizar los archivos db.php y functions.php
4. cada archivo de php con el nombre de la tabla, deberá de tener las funciones correspondientes para mostrar todos los
usuarios
5. En cada módulo deberá de existir una tabla html que muestre los registros que se encuentran en la tabla
correspondiente de la base de datos
6. Como lo más probable es que se dividan los módulos, entonces, el examen consistirá en modificar un módulo aleatorio
de acuerdo a lo que yo seleccione.
7. Si no logran modificar el módulo, por cada persona del equipo que no logre completar la actividad, el equipo completo
perderá 1 punto menos sobre la calificación final y la persona que no logró completar la actividad, perderá 2 puntos
menos sobre su calificación final.
8. No se podrá modificar la extensión del archivo dentro del módulo, es decir, no deberá de existir un index.php, solo
un index.html
9. Todos deberán de traer su computadora y tener el repositorio actualizado localmente para poder realizar la actividad
10. Todos deberán crear un pull request de su actividad para que el profesor pueda revisar su código y dar
retroalimentación
